Motivation.

I grew up in Singapore, and the issue of car park availability is something I’ve experienced firsthand. With the city’s high population density and government-regulated car ownership, finding a parking space can be a challenge, particularly during peak hours. This personal connection sparked my interest in using machine learning to predict parking availability.

As part of my Master's in Computer Science, I wanted to apply machine learning techniques to real-world problems, specifically one that had clear practical implications. I decided to focus on predicting car park availability in Housing Development Board (HDB) car parks, utilizing not only historical data but also contextual information like weather, public holidays, and the outcomes of Certificate of Entitlement (COE) bidding.

The goal of the project was to understand how these factors might influence parking demand and whether they could improve the accuracy of predictive models. This project gave me the opportunity to combine my knowledge of machine learning with a familiar, real-world issue that affects many people in Singapore.

How I Built It.

Language used: Python — perfect for machine learning tasks and data manipulation.

Main Components.

  • RF_parking.py: Trains and evaluates Random Forest models to predict the number of lots available in the next hour.
  • LSTM_parking.py: Builds, trains and evalutes LSTM models using Tensorflow/Keras, leveraging temporal sequences for evaluation.
  • dataset.py: Core data preperation script — connects to a PostgreSQL database, handles missing values, merges and scales contextual features, generates the target variable for supervised learning.
  • datasets/: Directory containing CSV files and the database schema used to create and populate tables in PostgreSQL.
  • requirements.txt: Specifies Python dependencies such as Pandas, Scikit-Learn, TensorFlow, and Psycopg.

The Logic.

  1. Data Processing:
    • Data from Singapore's open data portal was used, including car park lot data, weather forecasts, COE results and public holidays.
    • The data was preprocessed by aggregating minut-level car park availability into hourly intervals.
    • Contextual data like weather conditions, public holidays, and COE bidding results were integrated into the dataset.
  2. Model Development:
    • Baseline Model: Trained on historical parking lot data.
    • Contextual Model: Added external data (weather, public holidays, COE).
    • Used Scikit-Learn for Random Forest and TensorFlow for LSTM.
    • Tuned hyperparameters using Grid Search Cross Validation
    • Evaluated performance with metrics like MAE and RMSE.
  3. Final Models:
    • Random Forest: Slightly outperformed LSTM in terms of accuracy and required less computational resources.
    • LSTM: Good for time-series forecasting but was computationally heavier.

What I Learned.

  • Time-series forecasting: I gained hands-on expereince with LSTM, a model designed specifically for sequential data.
  • Machine learning in pracitce: learnt how to apply different algorithms to different real-world datasets and to critically evaluate their perfomance metrics.
  • Use of a databse: utilised PostgreSQL with many large datasets and managed it in SQL.
